# ADP 프로토타입 명세서

> 제칠일안식일예수재림교회 교인 교육 플랫폼 — 프로토타입(화면) 빌드용 단일 명세
> 이 문서는 "화면 흐름 + 정보 구조 검증"을 위한 프로토타입 기준이며, 무거운 로직(미션 자동배정·실시간 화상·실제 영상 추적)은 목(mock)으로 대체한다.

---

## 1. 개요

### 1.1 한 줄 정의
제칠일안식일예수재림교회의 신규·기존 교인 교육 플랫폼. 최상위 단위는 **ADP**, 운영 단위는 **교회**.

### 1.2 만들 면(surface) 3개
| 면 | 플랫폼 | 성격 | 쓰는 사람 |
|---|---|---|---|
| ① ADP 앱 | 모바일 | 보기 중심(학습 + 모니터링) | 교인 / 목회자·관리자 |
| ② 바이블가이드 앱 | 모바일 | 전도 도구 (ADP와 **별개 앱**) | 바이블가이드 |
| ③ ADP 관리자 페이지 | PC | 운영(제작·등록·권한) | 관리자 |

역할 분담의 큰 선: **PC = 운영(만들고·등록하고·권한 관리) / 앱 = 보기**.

### 1.3 프로토타입 범위 / 비범위
**범위(이번에 그리는 것)**
- 위 3개 면의 모든 화면 레이아웃과 화면 간 흐름
- 두 5단계 체계, 승급/매칭/전환 흐름이 화면상 드러나는 것
- 목 데이터로 채운 현실감 있는 화면

**비범위(이번엔 안 만드는 것 — 과구현 방지)**
- 1:1 화상의 실제 연결 → **외부 별도 서비스**가 담당, 앱은 레이아웃만
- 영상 재생 진행률 추적 → "완료" 버튼으로 단순화
- 퀴즈
- 조직 구조(연합회–합회–교회) 생성/편집 CRUD → 이미 짜여 있다고 전제
- 미션 자동배정 **로직** → 화면은 배정된 미션 1개를 목으로 표시
- 관심자(수강자) 입장 화면 → 바이블가이드 쪽 화면만

## 2. 공통 모델

### 2.1 계정 / 역할
**계정 타입(가입 시 결정, 둘 뿐)**
- 목회자
- 일반회원

**계정 위에 부여되는 권한·역할**
- **관리자 권한** — 레벨(교회 / 합회 / 연합회) 단일 축. `권한·조직 관리`에서 부여. 상위 레벨일수록 넓은 범위를 봄.
- **바이블가이드 역할** — 회원 중 지정. `바이블가이드 관리`에서 부여.

**기능적 상태**
- **교인** = ADP 교육에 등록되어 교육 단계를 가진 일반회원
- **관심자** = 아직 교인이 아닌 사람. 회원 계정이 아니라 **별도 데이터**(바이블가이드가 관리)

### 2.2 두 개의 5단계 체계 (서로 다른 체계 — 혼동 주의)
| | 교육 단계 (교인용, ADP) | 관심자 단계 (바이블가이드용) |
|---|---|---|
| 1 | 영혼구원 | 준비 |
| 2 | 정착 | 심기 |
| 3 | 교육 | 양육 |
| 4 | 지도자양성 | 추수 |
| 5 | 세계선교 | 보존 |

### 2.3 조직 / 권한
- 조직 계층: **연합회 > 합회 > 교회** (이미 존재, 프로토타입에선 보기용)
- 권한 범위: 교회 = 자기 교회 / 합회 = 산하 교회들 / 연합회 = 전체

### 2.4 핵심 엔티티
- **교인**: 이름, 현재 교육 단계, 단계 진도, 가입 경로(회원가입 / 수동등록)
- **관심자**: 이름(필수), 전화번호, 관계(복수 태그), 기념일(복수), 현재 관심자 단계, 담당 바이블가이드, 메모(비공개), 오늘의 미션, 교인 전환 여부
- **강의 패키지**: 교회별. 5단계 각각에 순서가 있는 강의 묶음
- **강의**: 제목, 영상 출처(유튜브 / 비메오 / 자체 업로드), 단계 내 순서, 완료 여부
- **미션**: 관심자별로 붙는 "오늘 이 사람을 위해 할 활동" 1건

## 3. 관통 규칙

### 3.1 교육 단계(교인) 승급
- 현재 단계의 **모든 강의 시청 완료 → 자동 승급**
- 강의는 **순차 잠금**: 앞 강의 완료해야 다음 해제
- 시청 완료 판정 = **"완료" 버튼**(프로토타입 단순화, 재생률 추적 없음)
- 교인은 **현재 단계 강의만** 노출(지난/다음 단계 안 보임)
- 시작 단계: 가입 시 **1.영혼구원 자동** / 관리자 수동 등록 시 시작 단계 지정 가능
- 관리자는 교인 단계를 **수동 조정**(승급/강등) 가능 → 자동 승급과 같은 단계 값을 건드림

### 3.2 관심자 단계 / 전환
- 관심자 단계 승급·강등은 **바이블가이드가 수동**(양방향)
- **관심자 → 교인 전환은 바이블가이드가 결정**
  - 정식 규칙: 보존(5) 단계에서만
  - 프로토타입: **전 단계에서 전환 버튼 노출**(메모로 표시)

### 3.3 매칭 (관심자 ↔ 바이블가이드) — 투웨이
- 바이블가이드가 관심자를 **직접 등록** → 자동으로 자기 담당
- 관리자가 `관심자 현황`에서 **담당 재지정** 가능
- **관심자가 데이터의 주인**: 담당이 바뀌어도 진행 기록(미션 이력·메모·단계)은 그대로, 담당자 꼬리표만 교체
- 바이블가이드는 **본인이 등록한 관심자만** 봄 / 관리자는 교회 전체 + 매칭을 봄
- 바이블가이드 역할 해제 시 담당 관심자는 **미배정**으로 풀림 → 관심자 현황에서 재지정

### 3.4 용어 규칙
- AI 관련 표현이 등장할 경우 **"알고리즘"**으로 표기(교단 제약)

### 3.5 빌드 메모
- 디자인은 **기존 디자인 시스템을 사용**(신규 디자인 시스템 생성 금지)
- 화면은 독립 컴포넌트로 먼저 만들고, `/preview` 한 페이지에 전 화면을 나열(인쇄 시 화면당 1페이지)

## 4. 면별 화면

> 각 화면은 동일한 5칸으로 기술한다: **진입 · 목적 · 보이는 것 · 행동 · 빈 상태/예외**

### ① ADP 앱 (모바일, 보기 중심)

#### 1-1. 회원가입 / 로그인
- **진입**: 앱 최초 실행 / 미로그인 상태
- **목적**: 계정 생성·로그인, 가입 시 **목회자 / 일반회원** 구분 확인
- **보이는 것**: 로그인 폼 · 가입 폼(계정 타입 선택 포함)
- **행동**: 로그인 → 역할에 맞는 첫 화면 / 가입 → 교인은 내 대시보드
- **빈 상태/예외**: 가입 시 목회자/일반회원 구분 검증 방식은 **미정**(자가 선택 vs 관리자 승인)

#### 1-2. 내 대시보드 (교인)
- **진입**: 교인 로그인 직후 첫 화면
- **목적**: 지금 단계에서 뭘 들어야 하는지 한눈에
- **보이는 것**: 현재 단계(가입 시 1.영혼구원 자동) · 현재 단계 진도율 · 이어보기(다음 강의) · 5단계 중 내 위치 표시 · **현재 단계 강의만 노출**
- **행동**: "강의 이어보기" → 강의 시청 / "강의 목록 보기" → 강의 목록
- **빈 상태/예외**: 배정 전 상태는 생략 / 현재 단계 모든 강의 완료 → 자동 승급 + 승급 완료 화면

#### 1-3. 강의 목록 (교인)
- **진입**: 내 대시보드 → "강의 목록 보기"
- **목적**: 현재 단계 패키지 강의를 순서대로 수강
- **보이는 것**: 현재 단계명 · 강의 카드(제목·길이·완료 여부) · 진도율 · **순차 잠금**(앞 강의 완료해야 다음 해제)
- **행동**: 잠금 해제된 강의 선택 → 강의 시청 (잠긴 강의는 비활성)
- **빈 상태/예외**: 강의 미등록 단계 → "준비 중"

#### 1-4. 강의 시청 (교인)
- **진입**: 강의 목록에서 선택 / 대시보드 이어보기
- **목적**: 강의 1편 시청 후 완료
- **보이는 것**: 영상 플레이어(유튜브/비메오/자체 업로드) · 강의 제목 · 다음 강의 안내
- **행동**: 재생 · **"완료" 버튼** → 다음 강의 잠금 해제 / 단계 마지막이면 자동 승급
- **빈 상태/예외**: 단계 마지막 강의 완료 → 승급 완료 화면

#### 1-5. 승급 완료 (교인)
- **진입**: 현재 단계 마지막 강의 완료 시 자동
- **목적**: 다음 단계 진입 안내·축하
- **보이는 것**: "○단계 완료! 다음은 △단계" · 새 단계 안내
- **행동**: "새 단계 강의 보기" → 강의 목록(다음 단계)
- **빈 상태/예외**: 5.세계선교 완료 시 → 전 과정 수료(이후 화면 **미정**)

#### 1-6. 우리교회 현황 대시보드 (목회자/관리자, 보기 전용)
- **진입**: 목회자·관리자 모드 로그인 직후
- **목적**: 우리 교회 교인들이 단계별로 어떤 상태인지 확인(외출 중 요약판)
- **보이는 것**: 단계별 인원 분포(영혼구원~세계선교) · 진척/참여 상태 · 권한 범위(교회>합회>연합회)에 따른 단위 차이
- **행동**: 보기 전용 (무거운 운영은 PC 관리자 페이지)
- **빈 상태/예외**: 데이터 없을 때 안내

---

### ② 바이블가이드 앱 (모바일, 별개 앱)

#### 2-1. 로그인
- **진입**: 앱 실행 / 미로그인
- **목적**: 바이블가이드 계정 로그인
- **보이는 것**: 로그인 폼
- **행동**: 로그인 → 메인 대시보드
- **빈 상태/예외**: —

#### 2-2. 메인 대시보드 (오늘의 미션)
- **진입**: 로그인 직후 첫 화면
- **목적**: 오늘 어떤 관심자에게 무엇을 해야 하는지 바로 확인
- **보이는 것**: **오늘의 미션 목록** — 관심자별 카드(이름 · 현재 단계 · "오늘 할 활동")
- **행동**: 미션 카드 탭 → 해당 관심자 상세 / 메뉴에서 관심자 목록·등록 진입
- **빈 상태/예외**: 오늘 미션 없음 → "오늘은 예정된 활동이 없어요" / 관심자 0명 → "첫 관심자를 등록하세요"
- **메모**: 단계별 분포·정체 알림·전환 수는 이번엔 제외(필요 시 추후 메인에 추가)

#### 2-3. 관심자 목록
- **진입**: 메인/메뉴 → "관심자 목록"
- **목적**: 내가 등록한 관심자 전체를 훑고 한 명 선택
- **보이는 것**: 전체 목록(필터 없음), **최근 등록순** · 카드마다 이름·현재 단계 · 상단 "관심자 등록"
- **행동**: 관심자 선택 → 상세 / "등록" → 관심자 등록
- **빈 상태/예외**: 0명이면 등록 유도

#### 2-4. 관심자 등록
- **진입**: 메인/목록 → "관심자 등록"
- **목적**: 새 관심자 한 명 등록
- **보이는 것**: 입력 폼 — 이름**(필수)** · 전화번호 · 관계(**복수 추가**, 가족/친구/동료/미분류) · 기념일(**복수 추가**, 날짜+종류) · 시작 단계 **준비** 자동
- **행동**: "+ 관계 추가" / "+ 기념일 추가"로 항목 늘림 · 저장 → 관심자 상세
- **빈 상태/예외**: 이름 비면 저장 막기 / 관계·기념일은 0개여도 저장 가능
- **메모**: 관계는 자유 복수 태그(예: 친구+동료)

#### 2-5. 관심자 상세 (핵심 화면)
- **진입**: 메인 미션 카드 / 관심자 목록에서 선택
- **목적**: 오늘 이 관심자를 위해 할 일을 수행하고 단계를 관리
- **보이는 것**(위→아래 우선순위):
  - **오늘의 미션** ← 최상단·가장 크게. 개인 특성에 맞춰 **자동 배정**된 활동 1건
  - 이름 + 현재 단계(간단히)
  - 나머지(전화·관계·기념일·메모)는 **더보기/접힘** 안쪽
- **행동**:
  - 미션 **수동 체크 → 완료**(프로토타입은 목 데이터)
  - 단계 **승급 / 강등**(양방향, 수동)
  - **교인 전환**(전 단계에서 가능 — 정식 규칙은 보존 단계만, 메모)
  - **1:1 화상** 시작 → 화상 화면
  - 메모 추가 · 정보 수정(더보기 안)
- **빈 상태/예외**: 오늘 미션 없음 표시 / 이미 교인 전환된 경우 상태 표시

#### 2-6. 1:1 화상
- **진입**: 관심자 상세 → "1:1 화상" 시작
- **목적**: 관심자와 얼굴 보며 동영상 함께 보기
- **보이는 것**: ① 각자 얼굴(양쪽 비디오 영역) · ② 함께 보는 동영상 화면 · ③ 채팅 — **이 3분할만**
- **행동**: (실제 연결·재생은 외부 별도 서비스) 프로토타입은 레이아웃만
- **빈 상태/예외**: 실제 화상은 외부 서비스 placeholder

---

### ③ ADP 관리자 페이지 (PC, 운영)

#### 3-1. 로그인
- **진입**: 관리자 페이지 접속 / 미로그인
- **목적**: 관리자 로그인
- **보이는 것**: 로그인 폼
- **행동**: 로그인 → 대시보드
- **빈 상태/예외**: 권한 없는 회원 접근 시 차단

#### 3-2. 대시보드 (통계)
- **진입**: 관리자 로그인 직후 첫 화면
- **목적**: 권한 범위 안에서 교육·전도 현황 한눈에
- **보이는 것**: 핵심 3지표 — 교인 **단계별 분포**(영혼구원~세계선교) · 관심자 **단계별 분포**(준비~보존) · **신규 유입 추이**. 권한 레벨(교회/합회/연합회)에 따라 보는 범위만 달라짐
- **행동**: 지표 클릭 → 해당 관리 화면으로
- **빈 상태/예외**: 데이터 없을 때 안내

#### 3-3. 강의 큐레이션 보드
- **진입**: 관리자 메뉴 → "강의 큐레이션 보드"
- **목적**: 5단계 각각에 들어갈 강의를 큐레이션(**교회별** 패키지)
- **보이는 것**: 한 화면에 **5단계 섹션을 세로로 스택**, 각 섹션 안에 강의 카드(제목·영상 출처)를 순서대로
  - 1.영혼구원 / 2.정착 / 3.교육 / 4.지도자양성 / 5.세계선교
- **행동**: 단계별 강의 추가(제목 + 유튜브/비메오/자체 업로드) · 순서 변경 · 편집 · 삭제
- **빈 상태/예외**: 강의 없는 단계 → 교인 앱에서 "준비 중"
- **메모**: 상위 관리자(합회/연합회)가 산하 교회 패키지를 다루는 방식은 권한 범위로 처리(이번 화면은 한 교회 기준)

#### 3-4. 교인 관리
- **진입**: 관리자 메뉴 → "교인 관리"
- **목적**: 교인 등록·조회 + 단계 진행 파악·조정
- **보이는 것**: 교인 목록 — 이름 · 현재 단계 · 진도 · 가입 경로(회원가입/수동등록) · 검색 · 상단 "교인 수동 등록"
- **행동**:
  - **수동 등록** — 시작 단계 **지정 가능**(기본 1.영혼구원, 임의 단계도)
  - 교인 선택 → 상세에서 **단계 직접 조정**(수동 승급/강등)
  - 검색
- **빈 상태/예외**: 0명이면 등록 유도

#### 3-5. 관심자 현황
- **진입**: 관리자 메뉴 → "관심자 현황"
- **목적**: 교회 전체 관심자 + 담당 바이블가이드 매칭을 보고 **조정**
- **보이는 것**: 관심자 목록 — 이름 · 현재 단계(준비~보존) · **담당 바이블가이드** · 교인 전환 여부 · 검색/단계 필터
- **행동**:
  - 관심자 선택 → 상세 보기
  - **담당 바이블가이드 지정/재지정**(투웨이 매칭)
  - 관리자가 관심자 추가·배정도 가능
- **빈 상태/예외**: 0명이면 안내
- **메모**: 담당 교체 시 관심자의 진행 기록(미션·메모·단계)은 유지, 담당자만 교체

#### 3-6. 바이블가이드 관리
- **진입**: 관리자 메뉴 → "바이블가이드 관리"
- **목적**: 교회 회원 중 바이블가이드 역할 지정/해제
- **보이는 것**: 바이블가이드 목록(이름 · 담당 관심자 수) · "바이블가이드 추가"
- **행동**: 추가 → 기존 회원 검색·선택해 역할 부여 / 해제 → 담당 관심자는 **미배정**으로 풀려 관심자 현황에서 재지정
- **빈 상태/예외**: 0명이면 안내

#### 3-7. 권한 / 조직 관리
- **진입**: 관리자 메뉴 → "권한/조직 관리"
- **목적**: 회원에게 관리자 권한 부여/회수 + 보는 단위(레벨) 지정
- **전제**: 조직 구조는 **이미 존재** → 조직 트리는 **보기용**(CRUD 없음)
- **보이는 것**: 조직 트리(보기) · 회원 목록 · 각 회원의 권한 레벨
- **행동**: 관리자 권한 부여/회수 + 레벨 지정(교회 / 합회 / 연합회). 같은 레벨 내 기능별 세분 없음(단일 축)
- **핵심 규칙**: 상위 레벨일수록 넓게 봄(교회 ⊂ 합회 ⊂ 연합회)
- **빈 상태/예외**: 권한 없는 회원 접근 시 차단
